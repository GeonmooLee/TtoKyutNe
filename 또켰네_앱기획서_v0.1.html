한 줄 정의
'또켰네'는 스마트폰을 오래 쓰는 문제보다, 짧은 간격으로 반복해서 다시 켜는 습관을 자각하게 만드는 앱입니다. 사용자가 화면을 다시 켠 순간을 포착하고, 이전 확인 이후 얼마나 시간이 지났는지 보여주어 “내가 왜 또 켰지?”를 부드럽게 돌아보게 합니다.

1. 프로젝트 개요

항목내용
앱 이름또켰네
카테고리생산성 / 디지털 웰빙 / 자기관리
1차 플랫폼Android Mobile
1차 출시 목표Google Play Store 내부 테스트 및 베타 배포
핵심 문제연락과 알림을 놓칠지 모른다는 불안으로 짧은 간격마다 폰을 다시 확인하는 습관
핵심 해결화면 재확인 간격을 숫자와 감성 문구로 보여주어 자각을 유도
MVP 원칙차단보다 자각, 통제보다 관찰, 서버보다 로컬 저장

2. 문제 정의

현대인은 카카오톡, DM, 이메일, SNS 알림 등으로 언제나 타인과 연결된 환경에서 살아갑니다. 이 초연결 환경은 빠른 소통과 편리함을 제공하지만, 동시에 “연락을 놓치면 안 된다”는 심리적 부담을 만듭니다.

사용자는 특별한 목적이 없어도 “혹시 답장이 왔을까?”, “내가 놓친 알림이 있지 않을까?”, “누군가 나를 찾고 있지는 않을까?”라는 불안감 때문에 반복적으로 화면을 켭니다. 이때 스마트폰은 필요한 순간에 쓰는 도구가 아니라, 불안을 해소하기 위해 습관적으로 확인하는 대상이 됩니다.

기존 디지털 웰빙 서비스는 하루 총 사용 시간, 앱별 사용량, 특정 앱 실행 횟수에 초점을 맞춥니다. 하지만 사용자가 실제로 겪는 문제는 단순히 오래 쓰는 것만이 아닙니다. 더 본질적인 문제는 몇십 초 혹은 몇 분 간격으로 현실의 흐름에서 계속 이탈한다는 점입니다.

핵심 문제 문장
현대인은 스마트폰을 오래 사용하는 것뿐만 아니라, 연락과 알림을 놓칠지 모른다는 불안으로 인해 짧은 간격으로 스마트폰을 반복 확인하며 현실에서 계속 이탈하고 있다.

3. 서비스 컨셉

컨셉 문장의미
재확인 간격을 보여주는 앱마지막 화면 켜짐 이후 몇 초/몇 분 만에 다시 켰는지 보여줌
불안을 숫자로 보여주는 앱무의식적 불안을 “37초 만에 다시 켬”처럼 관찰 가능한 지표로 전환
현실에서 이탈한 순간을 기록하는 앱집중, 휴식, 수면 흐름이 끊긴 순간을 데이터로 남김
연락 불안을 관찰하는 앱알림을 놓칠까 봐 반복 확인하는 행동을 부드럽게 인식시킴
어차피 지금은 아무 알림도 보이지 않습니다향후 알림 몰아보기 기능의 핵심 메시지

4. 타깃 사용자

사용자 유형상황앱이 주는 가치
알림 불안 사용자답장, DM, 업무 메시지를 놓칠까 봐 수시로 폰을 켬불안에 의한 재확인 행동을 숫자로 인식
학생공부 중 짧은 간격으로 폰을 확인해 집중이 끊김다시 켠 순간에 멈춰볼 계기 제공
직장인업무 중 카톡, 메일, 메신저 확인이 습관화됨업무 흐름 이탈을 줄이는 자기 관찰 도구 제공
자기관리 관심자스크린타임은 알지만 구체적으로 무엇을 고칠지 모름“얼마나 자주 다시 켰는지”라는 새로운 지표 제공

가치설명

8. v0.1 MVP 범위

MVP 원칙
v0.1의 목표는 “폰을 못 쓰게 막기”가 아니라 “다시 켠 순간을 알아차리게 하기”입니다. 사용자가 화면을 켠 사실, 이전 확인과의 간격, 그 순간의 감정적 질문을 확인할 수 있으면 MVP의 핵심은 달성됩니다.
구분포함 여부설명
화면 켜짐 감지포함사용자가 화면을 켠 시점 기록
재확인 간격 계산포함직전 화면 켜짐과 현재 화면 켜짐의 차이 계산
감성 문구 표시포함비난이 아니라 성찰을 유도하는 문구 표시
오늘 분석포함오늘 기준 핵심 지표만 표시
로컬 DB 저장포함서버 없이 기기 내부 저장
알림 몰아보기제외v0.3 이후 검토
LLM 문구 생성제외MVP에서는 로컬 문구 라이브러리 사용
로그인/동기화제외개인정보 보호와 개발 단순화를 위해 제외

9. 사용자 시나리오

상황사용자 감정앱 개입표시 문구 예시
공부 중집중하려 하지만 답장이 왔을지 궁금함10분 이내 재확인 시 알림 표시42초 만에 다시 켰어요. 지금 필요한 건 정보일까요, 안심일까요?
업무 중메일과 메신저를 놓치면 안 될 것 같음짧은 간격 반복 확인을 기록조금 전에도 확인했어요. 지금은 다시 일의 흐름으로 돌아가도 괜찮아요.
자기 전잠들기 전에도 새 알림이 궁금함밤 시간대 문구 표시오늘의 마지막 확인이어도 괜찮아요. 내일 봐도 늦지 않을 수 있어요.
쉬는 시간특별한 목적 없이 습관적으로 켬오늘 분석에서 반복 패턴 표시새 알림이 없어도 확인하고 싶을 때가 있죠. 그 마음을 알아차려봐요.

10. 감성 문구 방향

문구는 사용자를 혼내거나 죄책감을 주면 안 됩니다. “또 켰네”라는 이름은 자칫 꾸짖는 느낌을 줄 수 있으므로, 실제 문구는 부드러운 질문형과 자기 관찰형을 중심으로 설계합니다.

상황문구 예시
10분 이내 재확인37초 만에 다시 켰어요. 지금 필요한 건 정보일까요, 안심일까요?
10분 이내 재확인방금 확인했는데 다시 켰네요. 혹시 마음이 불안했나요?
5분 이내 재확인조금 전에도 확인했어요. 지금은 잠깐 쉬어도 괜찮아요.
밤 시간대오늘의 마지막 확인이어도 괜찮아요. 내일 봐도 늦지 않을 수 있어요.
자주 켠 날오늘은 폰을 자주 찾았어요. 바빴던 걸까요, 불안했던 걸까요?
집중 중지금 필요한 건 새 정보보다 다시 집중하는 힘일지도 몰라요.

11. 성공 지표

구분MVP 성공 기준
테스트 사용자지인 또는 초기 사용자 5-10명에게 설치 테스트
기능 정상 작동화면 켜짐 기록과 재확인 간격 계산이 실제 기기에서 정상 작동
자각 효과테스트 사용자 70% 이상이 “내가 자주 다시 켠다는 걸 알게 됐다”고 응답
문구 만족도테스트 사용자 70% 이상이 문구가 불쾌하지 않다고 응답
지속 사용3일 이상 앱을 켜둔 사용자가 50% 이상
데이터 변화3일 사용 후 10분 이내 재확인 횟수 감소 여부 관찰

12. 리스크와 대응

리스크설명대응
화면 켜짐 감지 안정성기기/OS 버전에 따라 백그라운드 이벤트 감지가 달라질 수 있음Android 우선 개발 후 실제 기기 중심 테스트
사용자 불쾌감“또켰네”가 혼나는 느낌을 줄 수 있음문구는 질문형, 공감형, 자기성찰형으로 설계
배터리 소모백그라운드 감지가 배터리에 영향을 줄 수 있음이벤트 기록만 최소 수행하고 불필요한 작업 제한
개인정보 우려사용 습관 데이터가 민감하게 느껴질 수 있음로그인 없음, 서버 없음, 로컬 저장, 삭제 기능 강조
기능 과다초기부터 알림 몰아보기까지 넣으면 개발 난이도 증가v0.1에서는 재확인 간격 자각에만 집중

13. 다음 액션 플랜

  • Android Studio에서 새 프로젝트를 생성하고 GitHub 저장소와 연결한다.
  • v0.1 MVP만 구현 대상으로 삼아 화면 켜짐 감지, 재확인 간격 계산, 로컬 저장부터 작업한다.
  • Codex에게 전체 구현을 한 번에 맡기지 말고, “프로젝트 구조 분석 -> 1개 기능 구현 -> 실행 확인 -> 오류 수정 -> Git 저장” 순서로 진행한다.
  • 실제 기기에서 화면 켜짐 감지와 알림 표시가 안정적으로 동작하는지 가장 먼저 검증한다.
  • 문구는 로컬 리스트 30-50개로 시작하고, LLM 문구 생성은 v0.2 이후 실험한다.
