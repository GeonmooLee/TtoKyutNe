3. 사용자 플로우

  1. 사용자가 앱을 설치하고 첫 실행한다.
  2. 온보딩에서 앱의 목적을 확인한다: “폰을 막는 앱이 아니라, 왜 또 켰는지 알아차리게 돕는 앱입니다.”
  3. 화면 켜짐 감지와 알림 표시를 위해 필요한 권한 안내를 확인한다.
  4. 사용자가 알림 조건을 설정한다. 기본값은 “10분 이내 다시 켰을 때만 문구 표시”이다.
  5. 사용자가 폰 화면을 끈 뒤 다시 켠다.
  6. 앱은 현재 화면 켜짐 시각을 기록하고, 직전 화면 켜짐과의 간격을 계산한다.
  7. 조건에 해당하면 감성 문구를 표시한다. ('직전 화면 켜짐과의 간격은 감성 문구와 상관없이 항상 표시된다)
  8. 사용자는 홈/오늘 분석 화면에서 오늘의 반복 확인 패턴을 확인한다.

5. 화면별 기능명세

5.1 온보딩 화면

항목명세
목적처음 사용자에게 앱의 정체성과 권한 필요성을 설명
주요 UI앱 소개 문구, 문제 정의 카드, 개인정보 보호 안내, 시작하기 버튼
사용자 액션시작하기 클릭, 권한 설정으로 이동, 건너뛰기
시스템 반응시작하기 클릭 시 권한 안내 화면 또는 홈 화면으로 이동
예외 상황권한 거부 시 “기능이 제한될 수 있습니다” 안내

5.2 홈 화면

항목명세
목적오늘의 반복 확인 상태를 가장 간단하게 보여줌
주요 UI오늘 화면 켠 횟수 카드, 마지막 재확인 간격 카드, 오늘의 한 줄 문구, 오늘 분석 버튼, 설정 버튼
표시 데이터today_screen_on_count, last_interval_seconds, shortest_interval_today
사용자 액션오늘 분석 보기, 설정 열기
시스템 반응로컬 DB에서 오늘 데이터를 집계해 카드 업데이트
예외 상황기록이 없으면 “아직 오늘 기록이 없어요” 표시

5.3 오늘 분석 화면

항목명세
목적하루 동안의 화면 재확인 패턴을 숫자로 확인
주요 UI총 화면 켠 횟수, 평균 재확인 간격, 최단 재확인 간격, 10분 이내 재확인 횟수, 시간대별 카드
계산 기준오늘 00:00 이후 screen_on_event 전체
사용자 액션기록 확인, 홈으로 돌아가기, 데이터 초기화 진입
시스템 반응집계값 계산 후 표시
예외 상황기록이 2개 미만이면 평균 간격 계산 불가 안내

5.4 설정 화면

항목명세
목적앱의 개입 강도와 데이터 관리를 사용자가 조정
주요 UI감성 문구 알림 on/off, 최소 알림 간격 선택, 야간 문구 사용 여부, 전체 데이터 삭제 버튼
기본값알림 사용 on, 최소 알림 간격 60초, 데이터 로컬 저장
사용자 액션설정 변경, 데이터 삭제, 권한 설정 열기
시스템 반응user_settings 업데이트
예외 상황데이터 삭제는 확인 다이얼로그 후 실행

6. 기능별 요구사항

ID기능요구사항완료 기준
F-01화면 켜짐 감지화면이 켜질 때마다 이벤트를 수신하고 현재 시각을 저장한다.실제 기기에서 화면을 켤 때마다 기록 생성
F-02간격 계산이전 screen_on_event와 현재 이벤트의 차이를 초 단위로 계산한다.첫 이벤트 예외 처리 및 이후 간격 정확도 확인
F-03문구 선택간격, 시간대, 최근 표시 이력을 기준으로 로컬 문구 중 하나를 선택한다.같은 문구가 과도하게 반복되지 않음
F-04알림 표시설정 조건을 만족하면 감성 문구를 상단 알림 또는 앱 내 카드로 표시한다.10분 이내 재확인 시 문구 표시
F-05오늘 분석오늘의 이벤트를 집계해 총 횟수, 평균, 최단, 10분 이내 횟수를 표시한다.앱 화면 값과 DB 기록이 일치
F-06설정 저장사용자 설정을 로컬에 저장하고 앱 재시작 후에도 유지한다.앱 재실행 후 설정 유지

7. 로컬 DB 구조 제안

7.1 screen_on_event

필드명타입설명
idInteger / PK고유 ID
screen_on_timeDateTime화면이 켜진 시간
previous_screen_on_timeDateTime nullable직전 화면 켜짐 시간
interval_secondsInteger nullable직전 켜짐과의 간격. 첫 기록은 null 가능
phrase_idInteger nullable표시된 문구 ID
created_atDateTime기록 생성 시간

7.2 phrase_history

필드명타입설명
idInteger / PK고유 ID
phrase_textString표시된 감성 문구
categoryStringshort_interval, night, frequent_day 등
shown_atDateTime문구 표시 시간
related_event_idInteger연결된 screen_on_event ID

7.3 user_settings

필드명타입설명

notification_enabledBoolean감성 문구 알림 사용 여부
min_interval_secondsInteger몇 초 이내 재확인일 때 알림을 띄울지. 기본값 600
quiet_hours_enabledBoolean야간 알림 제한 여부
data_retention_daysInteger데이터 보관 기간

8. Android 구현 고려사항

항목고려사항v0.1 방향
화면 켜짐 감지Android 버전과 제조사별 백그라운드 제한에 따라 동작 차이가 있을 수 있음실제 기기에서 이벤트 수신 우선 검증
백그라운드 동작앱이 꺼져 있거나 절전 모드일 때 동작 제한 가능불필요한 상시 작업을 피하고 이벤트 기록 중심으로 설계
알림 표시강제 팝업보다 사용자가 허용한 상단 알림이 안전알림 권한 안내 후 조건부 알림 표시
배터리상시 센싱처럼 보이면 배터리 우려 발생화면 켜짐 이벤트와 짧은 DB 기록만 수행
권한 안내사용자가 감시당한다고 느끼지 않도록 쉬운 설명 필요권한 요청 전에 목적과 로컬 저장 원칙 안내
iOS 대응화면 켜짐 감지와 알림 제어가 Android보다 제한적일 수 있음v0.1은 Android 우선, iOS는 별도 검토

9. 개인정보 보호 요구사항

원칙명세
로그인 없음v0.1에서는 회원가입, 소셜 로그인, 계정 기능을 제공하지 않는다.
서버 없음화면 켜짐 기록, 문구 이력, 설정은 모두 기기 내부에만 저장한다.
클라우드 동기화 없음사용 기록을 외부 서버로 전송하지 않는다.
알림 내용 미저장v0.3에서도 알림 본문 저장은 지양하고 앱 이름과 개수 중심으로 검토한다.
사용자 삭제권설정 화면에서 전체 기록 삭제 기능을 제공한다.
최소 권한MVP 동작에 꼭 필요한 권한만 요청한다.
사용자 안내 문구
또켰네는 화면을 다시 켠 간격을 계산하기 위해 화면 켜짐 상태를 감지합니다. 기록은 외부 서버로 전송되지 않고, 사용자의 기기 안에만 저장됩니다.

10. 감성 문구 라이브러리 초안

카테고리문구 예시
10분 이내37초 만에 다시 켰어요. 지금 필요한 건 정보일까요, 안심일까요?
10분 이내방금 확인했는데 다시 켰네요. 혹시 마음이 불안했나요?
5분 이내조금 전에도 확인했어요. 지금은 잠깐 쉬어도 괜찮아요.
밤오늘의 마지막 확인이어도 괜찮아요. 내일 봐도 늦지 않을 수 있어요.
자주 켠 날오늘은 폰을 자주 찾았어요. 바빴던 걸까요, 불안했던 걸까요?
집중 중지금 필요한 건 새 정보보다 다시 집중하는 힘일지도 몰라요.
알림 없음새 알림이 없어도 확인하고 싶을 때가 있죠. 그 마음을 알아차려봐요.

11. 개발 순서

순서작업완료 기준

12. 테스트 체크리스트

테스트 항목확인 내용
화면 켜짐 감지화면을 켤 때마다 기록이 저장되는가
첫 실행 예외첫 기록일 때 interval_seconds가 오류 없이 처리되는가
간격 계산30초, 1분, 5분 등 실제 간격과 계산값이 일치하는가
알림 조건설정한 최소 간격 이내일 때만 문구가 뜨는가
문구 중복같은 문구가 연속으로 과도하게 반복되지 않는가
오늘 분석총 횟수, 평균, 최단 간격, 10분 이내 횟수가 정확한가
데이터 삭제전체 기록 삭제 후 홈/분석 화면이 정상 상태로 돌아가는가
권한 거부권한이 없을 때 안내 화면이 뜨는가
배터리일상 사용 중 배터리 소모가 과도하지 않은가
불쾌감문구가 사용자를 비난하는 느낌을 주지 않는가

13. Codex 작업 요청 방식

작업 원칙
Codex에게 “전체 앱을 한 번에 만들어줘”라고 요청하지 말고, 한 번에 하나의 기능만 맡기는 방식이 안전합니다. 매 기능 구현 후 Android Studio에서 실행 확인하고, 문제가 없을 때 Git에 저장합니다.
단계Codex 요청 예시
프로젝트 분석이 Android 프로젝트 구조를 분석하고, 화면 켜짐 감지 기능을 추가하기 전에 수정해야 할 파일 목록을 알려줘. 아직 코드는 수정하지 마.
1개 기능 구현v0.1 PRD 기준으로 화면 켜짐 이벤트를 기록하는 기능만 구현해줘. 로컬 DB에는 screen_on_event를 저장해줘.
검증 요청방금 구현한 기능의 동작 흐름과 테스트 방법을 설명해줘. Android Studio에서 어떤 로그를 확인해야 하는지도 알려줘.
에러 수정아래 빌드 에러를 보고 원인을 설명한 뒤, 최소 수정으로 해결해줘. 다른 기능은 건드리지 마.
Git 저장이번 변경사항을 커밋하기 좋은 단위로 요약하고, 커밋 메시지를 제안해줘.

14. v0.1 완료 기준

  • 실제 Android 기기에서 화면 켜짐 이벤트가 기록된다.
  • 직전 화면 켜짐과 현재 화면 켜짐의 시간 차이가 초 단위로 계산된다.
  • 10분 이내 재확인 등 설정 조건에 맞을 때 감성 문구가 표시된다.
  • 오늘 분석 화면에서 총 횟수, 평균 간격, 최단 간격, 10분 이내 재확인 횟수가 표시된다.
